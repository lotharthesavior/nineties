# Database Documentation

## Overview

Arc uses **SQLite** as its database with **Diesel ORM** for type-safe database operations. This provides a lightweight, file-based database that requires no separate server process.

## Database Configuration

### Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `DATABASE_URL` | Path to SQLite database file | `database/database.sqlite` |
| `DATABASE_POOL_LIMIT` | Maximum connection pool size | `10` |

### Connection Pool

The application uses r2d2 connection pooling for efficient database access:

```rust
// src/helpers/database.rs
pub fn get_connection_pool() -> Pool<ConnectionManager<SqliteConnection>> {
    let database: String = env::var("DATABASE_URL")
        .unwrap_or_else(|_| "database/database.sqlite".to_string());
    let pool_limit: u32 = env::var("DATABASE_POOL_LIMIT")
        .unwrap_or_else(|_| "10".to_string())
        .parse()
        .expect("DATABASE_POOL_LIMIT must be a number");

    let manager = ConnectionManager::<SqliteConnection>::new(database);

    Pool::builder()
        .max_size(pool_limit)
        .test_on_check_out(true)
        .build(manager)
        .expect("Could not build connection pool")
}
```

The schema is auto-generated by Diesel CLI and stored in `src/schema.rs`:

```rust

        updated_at -> Nullable<Timestamp>,
    }

```

## Tables

| `created_at` | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | Creation timestamp |
| `updated_at` | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | Last update timestamp |

## Migrations

### Overview

Migrations are stored in the `migrations/` directory and embedded in the binary using Diesel's `embed_migrations!` macro.

```
migrations/
└── 2024-12-16-134059_create_users/
    ├── up.sql      # Migration forward
    └── down.sql    # Migration rollback
```

### Create Users Migration

**up.sql:**
```sql
CREATE TABLE IF NOT EXISTS users (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    name       TEXT NOT NULL,
    email      TEXT NOT NULL UNIQUE,
    password   TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

**down.sql:**
```sql
DROP TABLE users
```

### Running Migrations

```bash
# Run pending migrations
cargo run migrate

# Fresh migration (drops and recreates database)
cargo run migrate --fresh

# Migrate and seed
cargo run migrate --seed
```

### Creating New Migrations

Using Diesel CLI:

```bash
# Install Diesel CLI (if not installed)
cargo install diesel_cli --no-default-features --features sqlite

# Generate new migration
diesel migration generate create_posts
```

This creates:
```
migrations/
└── YYYY-MM-DD-HHMMSS_create_posts/
    ├── up.sql
    └── down.sql
```

### Embedded Migrations

Migrations are embedded in the binary for deployment:

```rust
// src/models/user.rs
pub const MIGRATIONS: EmbeddedMigrations = embed_migrations!();
```

Usage in `main.rs`:
```rust
conn.run_pending_migrations(models::user::MIGRATIONS)
    .expect("Failed to run migrations");
```

## Models

### User Model

```rust
// src/models/user.rs

#[derive(Queryable, Selectable, Debug)]
#[diesel(table_name = crate::schema::users)]
#[diesel(check_for_backend(diesel::sqlite::Sqlite))]
pub struct User {
    pub id: i32,
    pub name: String,
    pub email: String,
    pub password: String,
    pub created_at: Option<String>,
    pub updated_at: Option<String>,
}

#[derive(Insertable)]
#[diesel(table_name = crate::schema::users)]
pub struct NewUser<'a> {
    pub name: &'a str,
    pub email: &'a str,
    pub password: &'a str,
}
```

### Diesel Derives

| Derive | Purpose |
|--------|---------|
| `Queryable` | Allows struct to be loaded from database |
| `Selectable` | Enables compile-time column checking |
| `Insertable` | Allows struct to be inserted into database |

## CRUD Operations

### Create

```rust
use crate::schema::users::dsl::*;
use crate::models::user::NewUser;

let new_user = NewUser {
    name: "John Doe",
    email: "john@example.com",
    password: &prepare_password("secret123"),
};

diesel::insert_into(users)
    .values(&new_user)
    .execute(conn)?;
```

### Read

```rust
// Find by ID
let user: User = users.find(1).first::<User>(conn)?;

// Filter by email
let user: Vec<User> = users
    .filter(email.eq("john@example.com"))
    .load::<User>(conn)?;

// Select all
let all_users: Vec<User> = users.load::<User>(conn)?;
```

### Update

```rust
diesel::update(users.find(user_id))
    .set((
        email.eq("new@example.com"),
        name.eq("New Name")
    ))
    .execute(conn)?;
```

### Delete

```rust
diesel::delete(users.filter(email.eq("john@example.com")))
    .execute(conn)?;
```

## Seeders

### Seeder Trait

```rust
// src/database/seeders/traits/seeder.rs
pub trait Seeder {
    fn execute(conn: &mut SqliteConnection) -> Result<(), Box<dyn Error>>;
}
```

### User Seeder

```rust
// src/database/seeders/create_users.rs
pub struct UserSeeder;

impl Seeder for UserSeeder {
    fn execute(conn: &mut SqliteConnection) -> Result<(), Box<dyn Error>> {
        let expected_email: &str = "jekyll@example.com";

        // Check if already seeded
        let all_users: Vec<i32> = users.select(id).load::<i32>(conn).unwrap();
        if all_users.len() > 0 {
            println!("User table already seeded: {}", expected_email);
            return Ok(());
        }

        // Create default user
        diesel::insert_into(users).values(NewUser {
            name: "Jekyll",
            email: expected_email,
            password: &prepare_password("password"),
        }).execute(conn);

        Ok(())
    }
}
```

## Testing Database

### Test Configuration

Tests use a separate database configured via `.env.test`:

```
DATABASE_URL=database/test.sqlite
```

### Test Setup Pattern

```rust
fn prepare_test_db() -> PooledConnection<ConnectionManager<SqliteConnection>> {
    dotenv::from_filename(".env.test").ok();
    let mut conn = get_connection();
    conn.run_pending_migrations(MIGRATIONS).expect("Failed to run migrations");
    conn
}
```

### Test Cleanup

The `TestFinalizer` struct automatically cleans up the test database:

```rust
pub struct TestFinalizer;

impl Drop for TestFinalizer {
    fn drop(&mut self) {
        let database: String = env::var("DATABASE_URL")
            .unwrap_or_else(|_| "database/database.sqlite".to_string());
        let _ = std::fs::remove_file(database);
    }
}
```

### Transaction Testing

Use `test_transaction` for isolated tests:

```rust
conn.test_transaction::<_, Error, _>(|conn| {
    // Test code here
    // Changes are automatically rolled back
    Ok(())
});
```

## Database Location

```
database/
├── database.sqlite       # Production database
└── test.sqlite          # Test database (created during tests)
```

The `database/` directory includes a `.gitignore` to exclude database files from version control.

## Best Practices

1. **Use Connection Pool**: Always use `get_connection()` instead of creating direct connections
2. **Handle Errors**: Use `?` operator or explicit error handling for database operations
3. **Use Transactions**: Wrap related operations in transactions for atomicity
4. **Validate Input**: Sanitize user input before database operations
5. **Index Queries**: Add indexes for frequently queried columns
6. **Test with Isolation**: Use test transactions or separate test database
