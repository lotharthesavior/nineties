# PWA Implementation Analysis for Nineties

## Overview

This document explores the best options for implementing Progressive Web App (PWA) capabilities in Nineties, a Rust/Actix-web application using Hotwire (Turbo + Stimulus).

---

## What PWA Provides

| Feature | Benefit |
|---------|---------|
| Installable | Users can add to home screen, launches like native app |
| Offline Support | App works without network connection |
| Background Sync | Queue actions while offline, sync when online |
| Push Notifications | Re-engage users with server-sent notifications |
| Fast Loading | Service worker caches assets for instant loads |

---

## Architecture Options

### Option 1: Workbox (Recommended)

**What**: Google's library for generating service workers with best practices built-in.

**Pros**:
- Battle-tested, used by major sites
- Integrates with Vite via `vite-plugin-pwa`
- Configurable caching strategies
- Automatic cache versioning
- Precaching + runtime caching

**Cons**:
- Adds build complexity
- Learning curve for advanced customization

**Best for**: Production apps needing reliability and maintainability.

---

### Option 2: Vanilla Service Worker

**What**: Hand-written service worker without libraries.

**Pros**:
- Full control
- No dependencies
- Smaller file size
- Educational value

**Cons**:
- Must handle edge cases manually
- Cache invalidation is error-prone
- More code to maintain

**Best for**: Simple apps or learning purposes.

---

### Option 3: Hybrid (Recommended for Nineties)

**What**: Use `vite-plugin-pwa` for generation + custom service worker logic for Turbo integration.

**Pros**:
- Best of both worlds
- Handles Turbo Drive navigation properly
- Customizable offline page
- Easy manifest generation

**Cons**:
- Requires understanding both Workbox and service worker APIs

**Best for**: Nineties - we need Turbo-aware caching.

### 2. Caching Strategies

| Resource Type | Strategy | Rationale |
|---------------|----------|-----------|
| App Shell (HTML) | Network First | Always try fresh, fallback to cache |
| Static Assets (JS/CSS) | Cache First | Hashed filenames = immutable |
| Images | Stale While Revalidate | Show cached, update in background |
| API Responses | Network First | Data freshness matters |
| Fonts | Cache First | Rarely change |

### 3. Service Worker Structure

```javascript
// sw.js - Conceptual structure

// 1. Precache static assets (generated by Vite build)
precacheAndRoute(self.__WB_MANIFEST);

// 2. App shell - Network first with offline fallback
registerRoute(
  ({ request }) => request.mode === 'navigate',
  new NetworkFirst({
    cacheName: 'pages',
    plugins: [
      new CacheableResponsePlugin({ statuses: [200] }),
    ],
  })
);

// 3. Static assets - Cache first (already hashed)
registerRoute(
  ({ request }) => request.destination === 'script' ||
                   request.destination === 'style',
  new CacheFirst({
    cacheName: 'static-assets',
    plugins: [
      new ExpirationPlugin({ maxEntries: 50 }),
    ],
  })
);

// 4. Images - Stale while revalidate
registerRoute(
  ({ request }) => request.destination === 'image',
  new StaleWhileRevalidate({
    cacheName: 'images',
    plugins: [
      new ExpirationPlugin({ maxEntries: 100, maxAgeSeconds: 30 * 24 * 60 * 60 }),
    ],
  })
);

// 5. Offline fallback
setCatchHandler(async ({ event }) => {
  if (event.request.destination === 'document') {
    return caches.match('/offline.html');
  }
  return Response.error();
});
```

### 4. Turbo Drive Integration

Turbo Drive needs special handling because it:
- Fetches pages via `fetch()` with specific headers
- Expects HTML responses
- Manages its own page cache

**Solution**: Detect Turbo requests and handle appropriately:

```javascript
// Detect Turbo Drive requests
const isTurboRequest = (request) => {
  return request.headers.get('Turbo-Frame') !== null ||
         request.headers.get('Accept')?.includes('text/vnd.turbo-stream.html');
};

// For Turbo requests: Network first, no caching
registerRoute(
  ({ request }) => isTurboRequest(request),
  new NetworkFirst({
    cacheName: 'turbo-pages',
    networkTimeoutSeconds: 3,
  })
);
```

## Implementation Steps

### Phase 1: Basic PWA (Installable)

1. Add `vite-plugin-pwa` to project
2. Configure manifest in `vite.config.js`
3. Create app icons (192x192, 512x512, maskable)
4. Add manifest link to HTML head
5. Register service worker

**Effort**: ~2 hours

### Phase 2: Offline Support

1. Configure Workbox caching strategies
2. Create offline fallback page
3. Handle Turbo Drive requests
4. Test offline behavior

**Effort**: ~4 hours

### Phase 3: Advanced Features (Optional)

1. Background sync for form submissions
2. Push notifications
3. Periodic background sync
4. Share target API

**Effort**: ~8+ hours

## Vite Configuration

```javascript
// vite.config.js
import { defineConfig } from 'vite';
import { VitePWA } from 'vite-plugin-pwa';

export default defineConfig({
  plugins: [
    VitePWA({
      registerType: 'autoUpdate',
      includeAssets: ['favicon.ico', 'robots.txt', 'icons/*.png'],
      manifest: {
        name: 'Nineties App',
        short_name: 'Nineties',
        theme_color: '#4f46e5',
        background_color: '#ffffff',
        display: 'standalone',
        icons: [
          {
            src: 'icons/icon-192.png',
            sizes: '192x192',
            type: 'image/png'
          },
          {
            src: 'icons/icon-512.png',
            sizes: '512x512',
            type: 'image/png'
          },
          {
            src: 'icons/icon-maskable-512.png',
            sizes: '512x512',
            type: 'image/png',
            purpose: 'maskable'
          }
        ]
      },
      workbox: {
        globPatterns: ['**/*.{js,css,html,ico,png,svg,woff2}'],
        runtimeCaching: [
          {
            urlPattern: /^https:\/\/.*\.(?:png|jpg|jpeg|svg|gif|webp)$/,
            handler: 'StaleWhileRevalidate',
            options: {
              cacheName: 'images',
              expiration: {
                maxEntries: 100,
                maxAgeSeconds: 30 * 24 * 60 * 60
              }
            }
          }
        ],
        navigateFallback: '/offline.html',
        navigateFallbackDenylist: [/^\/api/, /^\/admin/]
      }
    })
  ]
});
```

## Customization Options

### 1. Cache Duration

```javascript
// Short-lived cache (news, feeds)
maxAgeSeconds: 60 * 60  // 1 hour

// Medium cache (user content)
maxAgeSeconds: 24 * 60 * 60  // 1 day

// Long cache (static assets)
maxAgeSeconds: 365 * 24 * 60 * 60  // 1 year
```

### 2. Update Strategy

```javascript
// Prompt user to update
registerType: 'prompt'

// Auto-update silently
registerType: 'autoUpdate'
```

### 3. Offline Scope

```javascript
// Cache everything
navigateFallback: '/offline.html'

// Exclude admin routes from offline
navigateFallbackDenylist: [/^\/admin/]

// Only cache specific routes
navigateFallbackAllowlist: [/^\/$/, /^\/public/]
```

### 4. Network Timeout

```javascript
// Fast timeout for poor connections
networkTimeoutSeconds: 3

// Patient timeout for slow APIs
networkTimeoutSeconds: 10
```

## Recommendation

**Start with Phase 1 + Phase 2** using `vite-plugin-pwa` with Workbox. This provides:

1. Installability (home screen icon)
2. Offline support for static assets
3. Offline fallback page
4. Automatic cache management
5. Turbo Drive compatibility

This covers 90% of PWA value with minimal complexity. Add Phase 3 features only if specific use cases require them.
